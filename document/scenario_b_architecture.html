Core Data Flow — Inventory Visualization & Aging Stock Identification
sequenceDiagram actor Manager as Dealership Manager participant UI as React Dashboard participant API as REST API participant Aging as Aging Service participant Vehicle as Vehicle Service participant Cache as Redis Cache participant DB as PostgreSQL Manager->>UI: Opens Inventory Dashboard UI->>API: GET /api/v1/vehicles?dealershipId=X API->>Vehicle: fetchVehicles(filters) Vehicle->>Cache: Check cached results alt Cache Hit Cache-->>Vehicle: Return cached data else Cache Miss Vehicle->>DB: SELECT vehicles WHERE dealership_id = X DB-->>Vehicle: Vehicle records Vehicle->>Cache: Store results (TTL: 5min) end Vehicle-->>API: Vehicle list API-->>UI: 200 OK + Vehicle JSON UI->>API: GET /api/v1/vehicles/aging?dealershipId=X API->>Aging: identifyAgingStock(dealershipId) Aging->>DB: SELECT * WHERE date_added < NOW() - 90 days DB-->>Aging: Aging vehicle records Aging-->>API: Aging stock list (with days_in_stock) API-->>UI: 200 OK + Aging Vehicle JSON UI-->>Manager: Display inventory + prominently highlight aging stock
Core Data Flow — Actionable Insights (Log Action for Aging Vehicle)
sequenceDiagram actor Manager as Dealership Manager participant UI as React Dashboard participant API as REST API participant Action as Action Service participant DB as PostgreSQL Manager->>UI: Clicks aging vehicle → "Log Action" UI->>UI: Opens Action Form Manager->>UI: Selects status / enters proposed action UI->>API: POST /api/v1/vehicles/{id}/actions
{status: "Price Reduction Planned", notes: "..."} API->>Action: createAction(vehicleId, payload) Action->>Action: Validate vehicle is aging (>90 days) Action->>DB: INSERT INTO vehicle_actions
(vehicle_id, status, notes, created_by, created_at) DB-->>Action: Action record persisted Action-->>API: 201 Created + Action record API-->>UI: 201 Created UI->>UI: Update vehicle card with new action status UI-->>Manager: Confirmation — action logged successfully
